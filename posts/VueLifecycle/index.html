Vue Lifecycle 이해하기

모든 Vue 앱은 Vue 함수로 새 Vue 인스턴스를 만드는 것부터 시작한다.

1
2
3

var vm = new Vue({
  // 옵션
})

Vue 인스턴스는 생성될 때, 일련의 초기화 단계를 거친다.
예를들어 아래와 같은 경우가 있다.

  • 데이터 관찰 설정이 필요한 경우
  • 템플릿을 컴파일 하는 경우
  • 인스턴스를 DOM에 마운트하는 경우
  • 데이터가 변경되어 DOM을 업데이트 하는 경우


이미지로 보는 라이프 사이클


1. Create 단계

라이프 사이클 훅에서 제일 처음 실행되는 Create 단계이다.
DOM이 생성되기 이전에 실행되는 훅으로써 DOM에 접근하거나 this.$el 을 사용하지 못한다.

이 단계는 beforeCreate 훅과 created 훅이 있다.

1-1. beforeCreate

create 단계에서도 먼저 실행되는 훅으로써 데이터와 이벤트, 감시자가 생성 되기 전에 호출된다.
따라서 이 훅에서는 data와 이벤트, 감시자등을 사용할 수 없다.

new Vue({
  render: h => h(App),
  data() {
    return {
      title: 'Vue.JS'
    };
  },
  beforeCreate () {
    console.log('제일 처음 실행되는 훅이다.')
    console.log(this.title)
    // data를 생성하기전에 호출됨으로 값은 undefined이다.
  }
}).$mount('#app')

data와 이벤트등을 사용할 수 없다

1-2. created

data, computed, methods, watch 등과 같은 옵션 설정이 완료된 시점으로 data와 이벤트, 감시자는 사용할 수 있다.

new Vue({
  render: h => h(App),
  data() {
    return {
      title: 'Vue.JS'
    };
  },
  created () {
    console.log('두번째 실행되는 훅이다.')
    console.log(this.title)
    // beforeCreate와 다르게 data를 생성이 후 호출하기 때문에 Vue.js의 값을 출력한다.
  }
}).$mount('#app')


2. Mount 단계

렌더링 직전에 컴포넌트에 직접 접근할 수 있는 단계이다.
초기 데이터를 설정할 때에는 created 훅을 사용하기를 권장한다.
그 이유로는 서버사이드렌더링을 지원하지 않기 때문이라고 한다.

이 단계는 beforeMount 훅과 mounted 훅이 있다.

2-1. beforeMount

템플릿과 렌더 함수들이 컴파일된 후에 첫 렌더링이 일어나기 직전에 실행된다.
렌더링이 되지 않았기 때문에 this.$el에 접근할 수 없다.

1
2
3
4
5
6
7

new Vue({
  render: h => h(App),
  beforeMount() {
    console.log(this.$el)
  }
}).$mount('#app')
// 아무것도 렌더링 되지않은 DOM만 출력된다.

2-2. mounted

컴포넌트, 템플릿, 렌더링된 돔에 접근할 수 있다.

1
2
3
4
5
6
7

new Vue({
  render: h => h(App),
  mounted() {
    console.log(this.$el)
  }
}).$mount('#app')
// 렌더링 된 DOM이 출력된다.

모든 컴포넌트가 마운트된 상태를 보장하지는 않는다.


3. Update 단계

컴포넌트에서 사용되는 데이터들이 변경되면 재 렌더링이 발생되면 실행된다.
디버깅을 위하여 재 렌더링 시점을 알고 싶을 때 사용할 수 있다.

이 단계는 beforeUpdate 훅과 updated 훅이 있다.

3-1. beforeUpdate

컴포넌트에서 사용되는 데이터들이 변경되고 Update 단계가 시작될 때 실행된다.
즉, 데이터 변경이 일어나는 시점과 랜더링을 하기 전에 실행된다.
재 랜더링이 되기 전이기 때문에 재 랜더링되는 DOM에는 접근할 수 없다.

3-2. updated

컴포넌트에서 사용되는 데이터들이 변경되고 Update 단계가 시작되고, DOM에 렌더링이 된 후 실행된다.
DOM이 재 랜더링이 된 후, DOM 접근이 가능한 훅이다.

// App.vue 파일

<template>
  <div id="app">
    
  </div>
</template>

<script>
export default {
  name: 'App',
  data() {
    return {
      testData : '데이터없음'
    }
  },
  mounted() {
    console.log('mounted');
    console.log(this.$el)
  },
  beforeUpdate() {
    console.log('beforeUpdate');
    console.log(this.$el)
  },
  updated() {
    console.log('updated');
    console.log(this.$el)
    this.$nextTick(function () {
      // 모든 화면이 렌더링된 후 실행한다.
    });
  }
}
</script>

What is nextTick function ❓
App.vue 파일의 28번째 라인을 보면 nextTick이라는 함수가 보인다.
data 가 업데이트 되고 난 직후, 비동기로 호출될 경우에 Vue가 DOM을 찾지 못하는 경우가 발생한다.
이 함수는 모든 호출이 완료된 후에 실행시켜줄 수 있는 함수이다. 위와 같은 상황을 방지해준다.

4. Destroy 단계

인스턴스를 해체할 때 호출한다.
이 단계는 beforeDestroy 훅과 destroyed 훅이 있다.

4-1. beforeDestroy

Vue 인스턴스 제거되기 직전에 호출된다.
제거되기 직전임으로 컴포넌트는 원래 모습과 모든 기능들을 그대로 가지고 있다.

4-2. destroyed

Vue 인스턴스 제거된 후에 호출된다.
Vue 인스턴스의 component에 걸려 있는 모든 이벤트가 해제된다.
모든 하위 Vue 인스턴스도 삭제된다.
